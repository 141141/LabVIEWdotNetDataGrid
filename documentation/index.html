Overview

 A simple data grid for use in place of LabVIEW's table that supports more of the standard expected table/grid functions for sorting, filters and auto fitting content. Basically, this grid is to allow better table support, with more built in features that you can use standard labview data with and basic properties to map the labview table to this one for supporting content.

Dependancies

 OpenG is used for several variant data inspectors

Usage

See the example code. Four simple steps to use it:
  1. Place a .NET DataGrid control on the front panel
  2. Initialize the DataGrid Class using the .net control reference
  3. Add all column definitions
  4. Add data as rows (in either string or cluster format)

Data Types

 Currently, there are several datatypes supported:
  • Strings
  • Booleans
  • Numbers
  • Timestamps
  • Images

Events
  • Cell Edit Ended - This single event is used currently as a sample for callback events.
  • Cell Value Changed - Works but errors currently on custom datatypes. Need to handle errors and extend this event
Adding new Events to the code
Events are created during initialization and registered as user events. in order to get the event structure to see the new events, the "registeredEvents" property needs to be updated with a new control when more are added to the array. This enables them to be visible to the event structure when opened from the class property.
